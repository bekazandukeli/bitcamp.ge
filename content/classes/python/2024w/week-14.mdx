#### პროექტი 102 - Blog API Project

შექმენით მარტივი REST API ბლოგის პლატფორმისთვის. API-ის უნდა შეეძლოს CRUD ოპერაციები ბლოგის პოსტებზე.

ბლოგის სტრუქტურა:
- title
- content
- published_date
- author
- სურვილისამებრ სხვა ველები

შექმენით სერიალაიზერები თქვენი მოდელისთვის, რათა გარდაქმნათ მონაცემები JSON-ში და პირიქით.
განსაზღვრეთ შესაბამისი url-ები და views-ები CRUD ოპერაციებისთვის.


#### პროექტი 103 - Creating Your Own Time Service

გარე სერვისების მოხმარება ერთია, მაგრამ მნიშვნელოვანია, რომ შეძლოთ საკუთარი სერვისის შექმნა,
რომლის გამოყენებაც თქვენც და სხვებსაც შეუძლიათ, რათა მხარი დაუჭიროთ სხვა დეველოპერებს,
რომლებსაც სურთ, გამოიყენონ სერვისები, რომლებსაც მათ სწორედ თქვენ მიაწვდით.
შექმენით მარტივი ვებ სერვისი, რომელიც აბრუნებს მიმდინარე დროს, როგორც json მონაცემებს.

მაგალითად:
```shell
"currentTime": "2050-01-24 15:06:26".
```

#### პროექტი 104 - Global Weather API

შექმენით REST API, რომელიც წამოიღებს ინფორმაციას ამჟამინდელ ამინდის პროგნოზზე.
ინფორმაცია ამინდზე უნდა მოდიოდიეს მსოფლიოს სხვადასხვა ლოკაციებიდან.

მთავარი ფუნქციონალი:
წამოიღეთ ინფორმაცია ამჟამინდელ ამინდზე კონკრეტული ქალაქიდან.
გამოიყენეთ ისეთი პარამეტრები, როგორიცაა ტემპერატურა, ტენიანობა, ქარის სიჩქარე და ა.შ.

უნდა შეგეძლოთ როგორც მოკლევადიანი ამინდის პროგნოზის წამოღება (24 საათი), ასევე გრძელვადიანის (7 დღე).
მომხმარებლებს უნდა შეეძლოთ ამინდის მოძიება ქალაქებისა და ქვეყნების მიხედვით.

აუცილებელი მოთხოვნები:
1. გამოიყენეთ Django REST Framework.
1. გამოიყენეთ რომელიმე ამინდის API კონკრეტულ ამინდზე ინფორმაციის წამოსაღებად და თქვენს API-ში იმპლემენტირებისათვის.


#### პროექტი 105 - Country Information API

შექმენით REST API, რომელიც წამოიღებს ინფორმაციას სხვადასხვა ქვეყნებზე.

- ქვეყნები:
თითოეული ქვეყნისთვის უნდა იყოს განსაზღვრული დედაქალაქი, მოსახლეობა, ფართობი და ოფიციალური ენები.
ასევე წამოიღეთ ისეთი გეოგრაფიული მონაცემები, როგორიცაა: კონტინენტი, მთავარი ქალაქები, კლიმატი და სხვა.
დამატებით შეგიძლიათ წამოიღოთ ქვეყანაზე ისეთი ინფორმაცია, როგორიცაა: GDP, ვალუტა და სხვა.

აუცილებელი მოთხოვნები:
1. გამოიყენეთ Django REST Framework.
1. ინფორმაცია ქვეყნებზე დაამეტეთ თქვენით და შეინახეთ მონაცემთა ბაზაში. არ არის აუცილებელი რომ ყველა ქვეყანა გქონდეთ შენახული, თუმცა, შეგიძლიათ გამოიყენოთ გარე, რომელიმე API, რომელიც ამ ინფორმაციას მოგაწვდით.
1. მომხარებელს უნდა შეეძლოს ქვეყნის id-ის ან ქვეყნის სახელის მიხედვით, კონკრეტულ ქვეყანაზე ინფორმაციის ამოღება.
