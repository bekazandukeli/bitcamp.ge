#### პროექტი 122 - CI Pipeline with Automated Tests

ნებისმიერი თქვენი პროექტისთვის, რომელიც github-ზე ატვირთული, შექმენით CI Pipeline-ი Github Actions-ს გამოყენებით.
თქვენს პროექტს ასევე უნდა ჰქონდეს გაწერილი ტესტების ფაილი. 
როდესაც რეპოზიტორიის მთავარ ბრენჩზე ახალი კომიტი დაიფუშება, უნდა ტრიგერდებოდეს CI, რომელიც:
- დააინსტალრებს requirements-ის ფაილს
- გაუშვებს ტესტებს
- გაუშვებს მიგრაციებს
- დასტარტავს პროექტს

#### პროექტი 123 - CI Pipeline with Docker 

აგრეთვე, ნებისმიერი თქვენი კონტეინერიზირებული პროექტისთვის, რომელიც გიტჰაბზეა ატვირთული, შექმენით CI Pipeline-ი Github Actions-ს გამოყენებით.
ყველაფერი იგივე უნდა ხდებოდეს, რაც 122-ე დავალებაში, უბრალოდ უნდა იქმნებოდეს Docker-ის Image-ი თქვენი დოკერიფაილი ბაზაზე და ის იტესტებოდეს.
გამოიყენეთ Github Secrets-ს, environment variables-ის ამოსაღებად და ჩასვით ისინი თქვენი პროექტის workflow-ში გიტჰაბზე.
